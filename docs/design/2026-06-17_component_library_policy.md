# 部品管理機能 方針

Date: 2026-06-17

## 目的

部品管理機能は、配管、バルブ、ポンプなどの標準部品を事前登録し、定常解析や非定常解析の画面で部品データから解析パラメータへ置換できるようにするためのマスタ管理画面とする。

## 基本方針

- 部品はカテゴリごとに共通メタデータと解析用パラメータを持つ。
- 初期カテゴリは `pipe`、`valve`、`pump` とする。
- 画面上の登録操作は、将来のDB保存APIに接続できるよう、部品レコード単位で完結した入力フォームにする。
- 解析画面では、部品IDまたは部品コードを参照して、登録済みパラメータをノードや配管リンクの入力値へ反映する。
- 登録済み部品は、解析ごとの一時設定ではなく、プロジェクト内で再利用可能なマスタデータとして扱う。

## データ方針

共通項目は次を持つ。

- 部品コード
- 部品名
- カテゴリ
- メーカー
- 型式
- タグ/用途メモ

カテゴリ別の初期パラメータは次を持つ。

- 配管: 形状、内径、長さ、粗さ、材質、定格圧力
- バルブ: 口径、Cv値、初期開度、圧力クラス、接続形式
- ポンプ: 定格流量、定格揚程、閉止揚程、効率、定格回転数

単位は画面に明示し、DB保存時には解析で使う標準単位へ変換する。初期実装ではフロント画面の入力単位をそのまま保持し、API実装時に変換境界を定義する。

## 解析画面との連携方針

- 定常流れ解析の配管、バルブ、ポンプノードから部品管理データを選択できるようにする。
- 部品を選択した場合、該当カテゴリのパラメータをノード入力値へコピーする。
- コピー後の解析条件は個別調整を許可する。マスタ側の値を直接変更しない。
- 将来的に「マスタ更新を既存解析へ再適用する」操作を追加できるよう、解析側には参照元部品IDを保持する。

## 画面方針

- サイドバーに「部品管理」タブを追加する。
- 画面は、左に部品一覧、右に登録/編集フォーム、下部または右側にカテゴリ別パラメータ確認を置く。
- 初期画面では、配管、バルブ、ポンプのサンプルデータを表示し、登録フローと一覧の形を確認できるようにする。
- DB APIが未実装の間は、フロントのローカル状態で登録できるひな型とし、API接続箇所を置き換えやすい構成にする。

## 今後の拡張

- 部品マスタのCRUD API。
- SQLiteまたはプロジェクトDBへの保存。
- CSVインポート/エクスポート。
- ポンプPQ曲線、バルブ開度-Cv曲線などのテーブル入力。
- 解析画面からの部品選択とパラメータ反映。
