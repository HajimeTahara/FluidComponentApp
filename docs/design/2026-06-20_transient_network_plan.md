# 非定常流れ解析への拡張メモ

## 背景

定常流れ解析は、圧力損失・ポンプ動作点・流体系統の成立性確認に価値がある。
一方、非定常解析は時間ステップ、状態量、初期条件、容量・慣性要素を扱うため、定常解析とは入力形式と結果表示が大きく異なる。

そのため、定常流れ解析を壊さず、別タブとして非定常流れ解析を育てる方針とする。

## UI方針

- `定常流れ解析` と `非定常流れ解析` は別タブにする。
- ただし、部品描画・接続操作・流体系統設定などの考え方は可能な範囲で共通化する。
- 定常側に `非定常解析へコピー` の導線を用意し、成立性確認済みのネットワークを非定常側へ渡せるようにする。
- 非定常側では、非対応部品は段階的に警告または無効化する。

## 初期MVP

液単相、密度一定、比熱・物性変化なしで開始する。

最初に扱う状態量は容量ノードの圧力とする。
ただし、以後の拡張を見据えて、実装はノード直結ではなくポート中心にする。

```text
dP/dt = net_q / C
```

- `P`: 容量ノード圧力
- `C`: コンプライアンス
- `net_q`: 接続ポートの流量収支

初期段階の抵抗要素は以下の最小式で扱う。

```text
q_a + q_b = 0
p_a - p_b = R q_a
```

配管・バルブ・エルボ・拡縮管などの詳細パラメータは一旦持ち込まず、まずはポート変数と部品方程式の管理を優先する。

各時刻では、圧力境界と容量ノードの圧力を既知値として扱い、抵抗要素で接続された未知接続点の圧力をネットワーク全体の連立方程式として解く。

```text
各未知接続点:
  sum(q_out) = sum(q_in_boundary)
```

これにより、`流量境界 - 抵抗 - 容量` のような接続でも、抵抗両端の圧力と流量をネットワーク全体の整合から決定できる。

## 段階的な拡張順

1. 容量ノードと圧力境界を実装する。
2. 既存の定常ネットワークを非定常タブへコピーできるようにする。
3. 準定常抵抗 + 容量ノードで圧力応答を時間積分する。
4. タンク液位、温度状態、熱容量、回転慣性を順に追加する。
5. 必要になったらDAE/陰解法へ拡張する。

## 注意点

OpenModelicaのような物理モデリングへ近づける場合、最終的には部品ごとに「ポート変数」と「状態変数」を分ける必要がある。

```text
params: 設計値・固定パラメータ
state: 時間発展する状態量
ports: 接続点の圧力・流量・温度・トルクなど
```

非定常タブでは、早い段階で以下の分離へ切り替える。

```text
component.params: 固定パラメータ
component.state: 時間発展する状態
component.ports: 接続点と物理量
equations: params/state/ports から作る部品方程式
```

実装上も、非定常ソルバは各ノードを一度 `component` に変換してから扱う。

```text
node
  -> component { params, state, ports, equations }
  -> connection graph
  -> equation assembly
  -> solver
```

初期の equation type は以下に絞る。

```text
pressure_constraint: p_port - p_set = 0
flow_source: q_port - q_set = 0
storage: C dp/dt - q_port = 0
tank_storage: A dh/dt - q_port = 0, p_port = rho g h
linear_resistance: p_a - p_b - R q_a = 0, q_a + q_b = 0
pipe_blasius: Darcy-Weisbach + Blasius friction model, q_a + q_b = 0
```

タンクの出口ポート高さはMVPでは 0 m とし、水位は `0 <= h <= h_max` にクランプする。
配管は円形または矩形断面を選べる。円形は内径、矩形は幅・高さから水力直径を求め、ブラジウス式で乱流摩擦係数を計算する。

MVPでは UI にすべてのポート物理量を表示しないが、バックエンドは各ポートの圧力・流量系列を返せる形にする。
