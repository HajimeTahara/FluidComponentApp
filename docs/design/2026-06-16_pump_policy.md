# ポンプ実装方針

## 目的

配管ネットワーク計算に、圧力損失を持つパイプとは逆に流体へ圧力を与えるポンプ要素を追加する。

## 基本方針

- ポンプは2ポート要素として扱う。
- 流れ方向は固定し、左ポートをIN、右ポートをOUTとする。
- ダイアグラム上の接続線の向きは物理方向として扱わない。ポンプ内部のIN/OUTだけを方向定義とする。
- ポンプは節点圧ソルバ内で、パイプと同じく「2つの圧力点を結ぶリンク」として扱う。

## PQ特性

初期実装では、閉止揚程、定格流量、定格揚程から二次曲線を作る。

```text
H(Q) = H0 - a Q^2
a = (H0 - Hr) / Qr^2
```

- `H0`: 閉止揚程 [m]
- `Qr`: 定格流量 [m3/h]
- `Hr`: 定格揚程 [m]
- `Qmax`: 最大流量 [m3/h]

圧力上昇は以下で計算する。

```text
boost_kPa = rho * g * H(Q) / 1000
```

## ソルバでの扱い

- ポンプ入口圧を `P_in`、出口圧を `P_out` とする。
- 要求昇圧 `P_out - P_in` に対応する流量をPQ特性から逆算する。
- 逆流は初期実装では許可しない。ポンプ流量は常に `IN -> OUT` の非負値とする。
- 要求昇圧が閉止昇圧を超える場合、流量は0とする。
- 要求昇圧が最大流量時の昇圧を下回る場合、流量は `Qmax` とする。

## 表示方針

- ポンプカードには `H0` と定格点を表示する。
- 計算後はポート横に入口圧、出口圧を表示する。
- ポンプカード内には昇圧量 `+ΔP` を表示する。
- 流量ラベルは既存のライン上ラベルに表示し、方向は `ポンプIN -> ポンプOUT` を基準にする。

## 今後の拡張

- 多点PQカーブ入力。
- 効率曲線と軸動力計算。
- 逆止弁なしポンプや逆流時抵抗のモデル化。
- ポンプ回転数変更による相似則。
